Rendered on Server This component is rendered on the server.. This component will be displayed to the users even before Angular runtime is bootstrapped in the browser. This component uses *browserExclude directive. Rendered in Browser This component is rendered in the browser by Angular runtime.. This component will be displayed to the users after Angular runtime gets bootstrapped in the browser. The server doesn't process it. This component uses *ssrExclude directive.

The above card case study: First server rendered version of the card component is displayed. And, when Angular runtime gets bootstrapped in the browser, it is replaced with browser rendered version. If you can't perceive the difference (because this demo is very small against your fast internet), disable the cache and throttle the speed to 2G or Slow 3G using your browser's dev tools.

You can also try performance profiling.
